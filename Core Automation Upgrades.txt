---
  Core Automation Upgrades (Free Tier Value)

  1. Smarter Scraping Pipeline
  - Multi-source deduplication — n8n compares new listings against existing DB, only processes truly new ones. Saves AI token costs.
  - Deadline-aware urgency queue — n8n sorts and prioritizes listings closing within 7 days, processes those first
  - Failed scrape retry logic — exponential backoff on failed sources, alert user if a source is consistently down

  2. Application Tracking Loop
  - n8n polls employer career portals after dispatch and updates status (applied → viewed → shortlisted → rejected) using HTTP requests
  - Currently the pipeline stops at "dispatched" — tracking closes the loop

  3. Smart Reminder System
  - n8n schedule: "You have 3 approved jobs sitting unsubmitted for 48hrs" → email nudge
  - Deadline countdown: "Chevening closes in 3 days — you haven't submitted" 
  - Weekly digest email: applications sent, responses received, upcoming deadlines

  Monetizable Feature Tiers

  Tier 1 — Pro Plan (~$15/mo)

  AI Interview Prep (n8n + AI)
  - After approval, n8n triggers a separate AI workflow: generates 10 likely interview questions based on the JD + user profile, plus model answers
  - Delivered to dashboard as a new "Interview Prep" tab on each job
  - n8n webhook receives "approved" event → calls AI → stores results

  Portfolio Match Scoring Improvement
  - n8n weekly job: re-scores all pending listings against an updated profile
  - User updates their CV → n8n detects profile change via webhook → triggers re-score across all open listings

  LinkedIn Job Sync (n8n HTTP node)
  - n8n polls LinkedIn Jobs API or scrapes authenticated session for personalized job recommendations
  - Unique because LinkedIn requires auth — gated behind Pro

  ---
  Tier 2 — Growth Plan (~$35/mo)

  Auto-Apply Pipeline (the full automation)
  - Currently: scrape → AI docs → email user → user approves → user manually applies
  - With n8n: after user approves, n8n uses Playwright/browser automation to actually submit the application to BrighterMonday, Fuzu, etc.
  - This is the core monetizable feature — "set it and forget it"
  - Implementation: n8n triggers POST /jobs/{id}/auto-apply → backend runs Playwright → status updates

  Scholarship Portal Auto-Fill
  - n8n + Playwright fills Chevening, CommonApp, DAAD portals
  - Uses profile data + generated docs
  - Massive time save — a Chevening application takes 8+ hours manually

  Email Response Handler
  - n8n monitors user's Gmail/Outlook via OAuth
  - Detects employer replies (interview invites, rejections, requests for more info)
  - Auto-categorizes and surfaces them in dashboard
  - Sends user a notification with suggested reply drafted by AI

  Tier 3 — Agency/Team Plan (~$99/mo)
  
  Multi-Profile Management
  - Recruitment agencies managing multiple candidates
  - n8n fan-out: one scrape run, matched and personalized per-candidate
  - Each candidate gets their own approval queue

  Bulk University Application
  - User selects 20 scholarship programs → n8n generates all 20 motivation letters in parallel (n8n splits items, calls AI for each in parallel branches)
  - Progress tracked in real-time in Tasks page

  White-label Outreach
  - n8n sends cold outreach emails to companies not actively hiring ("speculative applications")
  - AI drafts the email based on company research (n8n → Clearbit/Hunter.io for company data → AI → SMTP)
  - 5 emails/week on Pro, unlimited on Agency

 The api service has depends_on: postgres which is fine, but you just want to skip it. Run:

  docker compose up -d n8n postgres

  That starts only those two services. The api dependency check in n8n's config points to the internal http://api:5055 — since you're running the API locally, update the n8n env var so
   workflows can reach it. Add this to your .env:

  N8N_WEBHOOK_URL=http://localhost:5678

  And in any n8n workflows, use http://host.docker.internal:5055 as the API base URL instead of http://api:5055 — that's how Docker containers reach your locally-running Flask server
  on Linux (or set JH_API_BASE_URL=http://host.docker.internal:5055 in the compose env).

  To verify both are up:
  docker compose ps
